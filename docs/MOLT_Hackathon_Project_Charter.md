# MOLT Hackathon 项目立项说明

## 1. 项目基本信息

- 项目名称：`MOLT（脱壳）`
- 项目类型：`Hackathon Demo / 社交 x AI x 身份迁移`
- 立项日期：`2026-03-19`
- 项目周期：`2026-03-19 ~ 2026-03-20`

## 2. 立项目的

MOLT 旨在回应 AI 时代年轻人在专业、职业与身份认同上的失衡感。项目不是提供标准化求职建议，而是通过一条低门槛但有情绪承接的体验链路，帮助用户先被看见、再被定位、最后被连接。

Hackathon 阶段的目标不是构建完整社区，而是验证以下命题：

- 用户是否愿意通过三幕自白进入一个“非简历式”的表达空间
- “镜子确认”是否能让用户产生被理解感
- “压力区间 + Agent 一句话匹配”是否足以形成强记忆点

## 3. 目标用户

- 被 AI 冲击、开始怀疑自己专业价值的在校生和应届生
- 正在寻找迁移方向，希望看到相似人真实轨迹的探索者
- 已经完成转型，愿意以公开轨迹帮助后来者的前辈型用户

## 4. 核心问题

- 用户有情绪，但缺少不被审判的表达入口
- 用户看到大量信息，却看不到“像我这样的人后来去了哪里”
- 用户担心被替代，但难以区分结构性压力和可行动问题
- 用户想连接真实的人，但不想进入低质量社交

## 5. 产品假设

- 如果入口先承接情绪，而不是先要求填写简历，用户完成率会更高
- 如果系统使用用户自己的语言做镜子复述，用户会更容易继续完成后续流程
- 如果结果页同时给出压力区间和一条具体连接建议，用户会更容易感知产品价值

## 6. Hackathon 范围

- 必须跑通：
  - 三幕自白
  - 镜子确认
  - 压力区间结果
  - Agent 一句话匹配
- 加分项：
  - 能力重估卡片
  - 地图样例节点
  - 信号功能雏形

## 7. 成功标准

- 路演现场可在 `3 分钟` 内稳定跑完核心链路
- Demo 观众能在一次体验后复述产品核心句子
- 关键链路在 AI 失败时通过模板和样例数据继续演示

## 8. 风险与应对

- AI 调用链过长：收窄为单一 Happy Path，并为每个节点准备 fallback
- 外部数据不稳定：统一使用手工整理的岗位快照数据集
- 地图实现采用静态节点布局，不做实时图谱
- 体验过重：通过强叙事文案和单页流程控制认知负担

## 9. 路演表达重点

- 第一优先级：让评委理解“脱壳”这个隐喻
- 第二优先级：让评委看到情绪承接不是鸡汤，而是结构化连接的入口
- 第三优先级：再展示地图、节点、信号这些视觉加分层
